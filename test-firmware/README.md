# Test Firmware for Flexers

This directory contains minimal test firmware for validating the Flexers ESP32 emulator.

## Files

- `minimal.S` - Assembly source code for minimal test firmware
- `build.sh` - Build script (requires ESP32 toolchain)
- `minimal_test.bin` - Pre-built test binary in ESP32 format (hand-crafted)

## Test Firmware Overview

The minimal test firmware validates:
1. **Basic execution** - Xtensa instruction execution from flash
2. **ROM function calls** - Calling ROM stubs (ets_get_cpu_frequency, ets_printf, etc.)
3. **Memory operations** - Stack usage, data access
4. **Halt detection** - Infinite loop at end

## Memory Layout

- **Code section**: 0x40080000 (Flash instruction region)
- **Data section**: 0x3FFA0000 (SRAM)
- **Stack**: Top of SRAM (0x3FFFFFFF)

## Building from Source

If you have the ESP32 toolchain installed:

```bash
cd test-firmware
chmod +x build.sh
./build.sh
```

Requirements:
- xtensa-esp32-elf-gcc
- esptool.py

## Using Pre-built Binary

For testing without toolchain:

```bash
cargo run --example firmware_loader test-firmware/minimal_test.bin
```

Or in tests:

```rust
let firmware = include_bytes!("../test-firmware/minimal_test.bin");
```

## Binary Format

ESP32 binary format (as generated by esptool.py):

```
Offset | Size | Field
-------|------|-------
0      | 1    | Magic (0xE9)
1      | 1    | Segment count
2      | 1    | SPI mode
3      | 1    | SPI speed/size
4      | 4    | Entry point (little-endian)
8+     | N    | Segments
```

Each segment:
```
0      | 4    | Load address
4      | 4    | Data length
8      | N    | Data bytes
```
