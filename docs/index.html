Why Flexers?

Built for the Modern Rust Ecosystem

⚡

Zero FFI Overhead

Pure Rust implementation eliminates FFI marshalling costs. Direct memory access and native function calls.

🛡️

Memory Safe

No null pointers, no use-after-free, no data races. Rust's ownership system prevents entire classes of bugs.

🔧

Easy Integration

Standard Cargo crate. No manual bindings, no build scripts for C compilation, no platform-specific hacks.

🐛

Better Debugging

Full backtraces, thread-safe by design, clear error types. No pointer aliasing mysteries.

📦

Modular Design

Clean separation: flexers-core (CPU), flexers-periph (I/O), flexers-stubs (ROM), flexers-session (API).

⚙️

Performance First

Page-table memory lookup, inline fast paths, cache-aligned CPU state, compiler jump table dispatch.

Development Timeline

Complete IoT Emulation Platform

Phases 1-4: Foundation ✅ 100%

Core CPU, Memory, Peripherals, Flash SPI

  • ✅ Xtensa CPU with windowed registers
  • ✅ Page-table memory (520KB SRAM, 448KB ROM, 4MB flash)
  • ✅ UART, timers, GPIO, interrupt controller
  • ✅ SPI flash emulation with write protection

Phases 5-7: Advanced Peripherals ✅ 100%

ADC, DAC, PWM, I2C, DMA, Touch, RMT

  • ✅ 8-channel 12-bit ADC
  • ✅ 2-channel 8-bit DAC
  • ✅ 16-channel PWM (LEDC)
  • ✅ I2C master/slave (2 instances)
  • ✅ DMA with descriptor chaining
  • ✅ 10-channel capacitive touch sensing
  • ✅ 8-channel RMT (IR, WS2812 LEDs)

Phases 8-9: FreeRTOS ✅ 100%

Complete RTOS emulation for multitasking

  • ✅ Task scheduler with priority preemption
  • ✅ Queues, semaphores, mutexes
  • ✅ Event groups for task synchronization
  • ✅ Software timers with callbacks
  • ✅ Timer interrupt integration (automatic scheduling)
  • ✅ 295+ FreeRTOS tests passing

Phase 10A: Real Network I/O ✅ 100%

Host socket bridging for real network communication

  • ✅ Socket manager (fake FD → real host sockets)
  • ✅ TCP client/server with non-blocking I/O
  • ✅ UDP send/receive
  • ✅ Real DNS resolution via std::net
  • ✅ Connect to AWS IoT, MQTT brokers, HTTP servers
  • ✅ 260+ tests passing (100% pass rate)

Phase 10B-D: Advanced Networking

In Progress • Enhanced network features

  • select()/poll() for socket multiplexing
  • TLS/SSL support (rustls)
  • IPv6 dual-stack
  • Socket options (TCP_NODELAY, etc.)
  • Async I/O with background threads
  • Network simulation (latency, packet loss)

Future: lwIP & JIT

Optional enhancements

Design Optimizations

Component Optimization Impact
Memory Access Page table + inline fast path O(1) lookup, no bounds checks
CPU State 64-byte cache alignment Hot registers in single cache line
Dispatch Nested match statements Compiler jump tables (vs function pointers)
Integration Zero FFI overhead Direct Rust function calls
